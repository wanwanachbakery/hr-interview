## 1. ระบบนี้คืออะไร

HR-WWN เป็นระบบ HR ที่ทำ 2 อย่าง:

1. **เก็บข้อมูลพนักงาน** — ฝ่าย / แผนก / ตำแหน่ง / รายชื่อ
2. **AI สัมภาษณ์งานพนักงาน** — ตอบคำถามรายวัน → AI ออกเอกสาร JD (รายละเอียดงาน) + KPI + ข้อเสนอปรับปรุงให้

เหมาะกับบริษัทเล็ก-กลาง ที่ต้องการ:
- มีคลัง JD/KPI ของพนักงานทุกคนแบบ updateได้
- รู้ว่าเวลาในแต่ละชั่วโมง พนักงานทำงานอะไรอยู่
- หาจุดปรับปรุง / งานที่ AI ช่วยได้

## 2. คอมที่ใช้ต้องมีอะไรบ้าง

### Hardware
- คอม Windows 10 หรือ 11 (Mac/Linux ก็ได้ แต่คู่มือนี้เน้น Windows)
- RAM 4 GB ขึ้นไป
- Disk ว่าง 1 GB ขึ้นไป

### Software ที่ต้องติดตั้ง
มี **1 อย่าง** ที่ต้องลงเพิ่ม: **Node.js**

ส่วนตัว HR-WWN เองไม่ต้องลง — แค่ดับเบิลคลิกไฟล์ INSTALL ระบบจัดการให้

> **เคล็ดลับ**: คอมต้องเปิดตลอดที่ต้องการให้คนใช้งาน — ถ้าปิด/sleep = ระบบดับ

## 3. ติดตั้งครั้งแรก

ทำขั้นนี้แค่ครั้งเดียว เปลี่ยนเครื่องใหม่ค่อยทำใหม่

### ขั้นที่ 3.1 — เปิดโฟลเดอร์โปรแกรม
ไปที่โฟลเดอร์ที่เก็บไฟล์ HR-WWN ไว้ (ที่ทีม IT ให้มา หรือที่ unzip ไว้)

จะเห็นไฟล์ประมาณนี้:
```
📁 Hr-wwn
   ├── 📄 อ่านก่อน.txt
   ├── 📄 คู่มือใช้งาน.md
   ├── ⚙️ 1-INSTALL.bat
   ├── ⚙️ 2-START.bat
   ├── ⚙️ 3-BACKUP.bat
   ├── ⚙️ 4-TUNNEL.bat
   ├── 📁 public
   ├── 📁 data
   └── ... (ไฟล์อื่นๆ)
```

### ขั้นที่ 3.2 — ดับเบิลคลิก `1-INSTALL.bat`

หน้าต่างสีดำจะเปิดขึ้น

**กรณีที่ 1: ไม่มี Node.js บนเครื่อง**
- จะเห็นข้อความ `[X] ไม่พบโปรแกรม Node.js`
- หน้าเว็บ https://nodejs.org/ จะเปิดให้
- ที่หน้าเว็บนั้น คลิกปุ่ม **LTS** (สีเขียว) ดาวน์โหลด
- ดาวน์โหลดเสร็จ → ดับเบิลคลิกไฟล์ .msi → กด **Next** ไปเรื่อยๆ จนจบ → กด **Finish**
- กลับมาดับเบิลคลิก `1-INSTALL.bat` อีกรอบ

**กรณีที่ 2: มี Node.js อยู่แล้ว**
- จะเห็น `[OK] พบ Node.js แล้ว`
- ระบบจะติดตั้งแพ็คเกจที่จำเป็น (ใช้เวลา 1-3 นาที)
- พอเสร็จจะเห็น `[OK] ติดตั้งเสร็จเรียบร้อย!`

กดปุ่มใดก็ได้เพื่อปิดหน้าต่าง

## 4. เริ่มใช้งาน

## 5. Login ครั้งแรก + เปลี่ยนรหัส

หน้าแรกที่เห็น = หน้า Login

กรอก:

- **Password**: `JC2026!Init`

## 6. ตั้งค่าบริษัท

ก่อนสร้าง user ต้องตั้งโครงสร้างองค์กรก่อน

### ขั้นที่ 6.1 — ตั้งชื่อบริษัท
1. ที่หน้า Admin Console → คลิกการ์ด **🏢 โครงสร้างองค์กร**
2. กรอกชื่อบริษัท (TH/EN) → กด **บันทึก**

### ขั้นที่ 6.2 — สร้างฝ่าย
ในหน้าเดิม (โครงสร้างองค์กร) มีกล่อง "เพิ่มฝ่ายใหม่"
- ชื่อฝ่าย: เช่น `ฝ่ายขาย`
- Icon (emoji): `💰` หรือทิ้งไว้เป็น `🏢`
- สี: เลือกตามใจ
- กด **+ เพิ่มฝ่าย**

ทำซ้ำตามฝ่ายที่บริษัทมี เช่น `ฝ่ายผลิต` `ฝ่ายบัญชี` `ฝ่ายไอที`

### ขั้นที่ 6.3 — สร้างแผนกในฝ่าย
ในกล่องของแต่ละฝ่าย จะมีฟอร์ม "เพิ่มแผนกในฝ่ายนี้"
- ใส่ชื่อแผนก เช่น `แผนกขายหน้าร้าน`
- กด **+ เพิ่มแผนก**

### ขั้นที่ 6.4 — สร้างตำแหน่งในแผนก
ในกล่องของแต่ละแผนก จะมีฟอร์ม "เพิ่มตำแหน่งในแผนกนี้"
- ใส่ชื่อตำแหน่ง เช่น `หัวหน้าทีมขาย`
- กด **+ เพิ่มตำแหน่ง**

> **เคล็ดลับ**: สร้าง position ให้ครบทุกตำแหน่งในบริษัทก่อน ค่อยสร้าง user
> เพราะตอนสร้าง user ต้องเลือก ฝ่าย/แผนก/ตำแหน่ง ครบ 3 ฟิลด์

## 7. สร้างผู้ใช้งาน (User)

### ขั้นที่ 7.1 — ไปหน้าจัดการ user
- กลับไปหน้า Admin Console (กดเมนูบน "🔐 Admin")
- คลิกการ์ด **👥 จัดการผู้ใช้**

### ขั้นที่ 7.2 — กด **+ เพิ่มผู้ใช้ใหม่**

กรอก:
- **Username**: ภาษาอังกฤษล้วน เช่น `somchai`
- **รหัสผ่าน**: ตั้งให้พนักงาน
- **ชื่อ-นามสกุล**: ชื่อจริง
- **Role**: เลือกระดับสิทธิ์
  - **ผู้บริหาร** — เห็นทั้งบริษัท
  - **ผู้จัดการ** — เห็นเฉพาะฝ่ายตัวเอง
  - **หัวหน้าฝ่าย** — เห็นฝ่ายตัวเอง
  - **หัวหน้าแผนก** — เห็นแผนกตัวเอง
  - **เจ้าหน้าที่** — เห็นเฉพาะตัวเอง
- **ฝ่าย / แผนก / ตำแหน่ง**: เลือกที่พนักงานอยู่ (ต้องครบทั้ง 3)
- **เวลาทำงาน**: ค่าเริ่มต้น 09:00–18:00 + พัก 12:00–13:00 (แก้ได้)

กด **บันทึก**

### ขั้นที่ 7.3 — แจกรหัสให้พนักงาน
หน้านี้แสดงรายการ user ทั้งหมด แต่ **ไม่แสดงรหัสผ่าน** (เพราะถูก hash) — คุณต้องจดรหัสที่ตั้งไว้ส่งให้พนักงานเอง

> 💡 ระบบสร้าง **employee record** ให้อัตโนมัติทันทีที่สร้าง user — พนักงาน login มาแล้วเห็นการ์ด "🎤 อินเทอร์วิวของฉัน" รออยู่เลย

## 8. ให้พนักงานเริ่มตอบ Interview

### พนักงานทำเอง (คุณส่งคู่มือนี้ให้พนักงาน):

1. เข้า URL (ที่คุณส่งให้ — `http://localhost:3000` ถ้าใช้ในออฟฟิศ)
2. Login ด้วย username/password ที่ได้รับ
3. หน้าแรกจะเห็นการ์ด **🎤 อินเทอร์วิวของฉัน — รอตอบ**
4. กดปุ่ม **▶ เริ่มอินเทอร์วิว**
5. ระบบจะถามให้เลือก:
   - ภาษา (ไทย)
   - กะทำงาน (09-18 หรือ 10-19)
   - วันที่สัมภาษณ์
6. กด **เริ่มอินเทอร์วิว →**
7. AI ถามคำถามทีละข้อ — ตอบในกล่องข้อความ กด Enter ส่ง
8. ตอบครบประมาณ 15 ข้อ → กด **สร้างเอกสาร →**
9. ได้ JD + KPI + Optimization ดูที่หน้า Review

> ⏱️ ใช้เวลาประมาณ 15-20 นาทีตอบครบ
> สามารถปิดเว็บแล้วกลับมาตอบต่อทีหลังได้ — ระบบจำคำตอบเก่าให้

### หัวหน้าดูสถานะลูกน้อง

- หัวหน้าฝ่าย/แผนก login → กด **📋 Dashboard** บนเมนู
- เห็น tree ของทุกตำแหน่งในขอบเขต พร้อมสถานะ:
  - 🟢 **เสร็จแล้ว** — ตอบครบ + มีเอกสาร JD/KPI
  - 🔵 **กำลังตอบ** — ตอบบางส่วน
  - 🟡 **ยังไม่ตอบ** — ยังไม่ได้เริ่ม
  - ⚪ **ตำแหน่งว่าง** — ยังไม่มี user

## 9. Backup ข้อมูล

**ทุกสิ่งทุกอย่างเก็บใน `data/` folder** — ถ้าไฟล์หาย = ข้อมูลพนักงานหาย

### วิธีง่ายสุด:
ดับเบิลคลิก `3-BACKUP.bat` — ระบบจะคัดลอก data ไป folder `backup/YYYY-MM-DD_HH-MM/`

### แนะนำ:
- ทำ backup **อย่างน้อยสัปดาห์ละครั้ง**
- ทุกเดือนคัดลอกโฟลเดอร์ `backup/` ทั้งหมดไปไว้ที่:
  - **Google Drive** / OneDrive (ฟรี)

  - คอมเครื่องอื่น

> ⚠️ **อย่าเก็บ backup ไว้บนคอมเครื่องเดียวกัน** ถ้า disk เสีย backup หายด้วย!

## 10. เปิดให้คนนอกเข้าใช้ผ่าน Internet

ถ้าพนักงานทำงานนอกออฟฟิศ ต้องเปิดให้เข้าผ่าน Internet

### วิธีง่ายสุด: Cloudflare Tunnel (ฟรี)

#### ขั้นที่ 10.1 — ติดตั้ง cloudflared

เปิด **PowerShell แบบ Administrator** (คลิกขวาที่ Windows logo → Terminal (Admin))

พิมพ์คำสั่งนี้แล้วกด Enter:
```
winget install Cloudflare.cloudflared
```

รอติดตั้งเสร็จ (1-2 นาที)

#### ขั้นที่ 10.2 — เริ่ม tunnel

1. เปิด `2-START.bat` ให้เซิร์ฟเวอร์รันอยู่
2. **อย่าปิดหน้าต่างนั้น** — เปิดอีก 1 หน้าต่าง
3. ดับเบิลคลิก `4-TUNNEL.bat`
4. รอสักครู่ จะเห็น URL ขึ้น เช่น:
   ```
   https://something-random.trycloudflare.com
   ```
5. ส่ง URL นี้ให้พนักงาน — เข้าจากที่ไหนก็ได้

> ⚠️ **URL จะเปลี่ยนทุกครั้งที่ปิด-เปิดใหม่** — ถ้าอยากให้ URL คงที่ ต้องมี domain (อ่านหัวข้อ 10.5 ด้านล่าง)

> ⚠️ **PC ของคุณต้องเปิดตลอด** ถ้าปิด/หลับ = พนักงานเข้าไม่ได้

### 10.5 ใช้ domain ของตัวเอง (URL ถาวร เช่น `hr.mycompany.com`)

ถ้ามี domain อยู่แล้ว ไม่ต้องใช้ URL ของ Cloudflare ที่เปลี่ยนทุกครั้ง

#### ขั้นที่ 1 — ย้าย domain ไป Cloudflare (ถ้ายังไม่ได้ย้าย)

**ถ้า domain คุณอยู่ที่ Cloudflare แล้ว** → ข้ามไปขั้นที่ 2 เลย

**ถ้าอยู่ที่อื่น** (GoDaddy, Namecheap, Route53, ฯลฯ):
1. สมัคร Cloudflare ฟรี: https://dash.cloudflare.com/sign-up
2. คลิก **Add a site** → ใส่ domain ของคุณ → เลือกแผน **Free**
3. Cloudflare จะให้ **2 nameserver** เช่น:
   - `ada.ns.cloudflare.com`
   - `bob.ns.cloudflare.com`
4. เข้าหน้า registrar ของคุณ (ที่ซื้อ domain) → หาเมนู **Nameservers** หรือ **DNS**
5. เปลี่ยน nameserver เป็น 2 ตัวที่ Cloudflare ให้
6. รอ 5 นาที - 24 ชั่วโมง (ปกติประมาณ 30 นาที) ให้ Cloudflare detect
7. Cloudflare จะส่ง email ยืนยันเมื่อพร้อม

**ค่าใช้จ่าย**: ฟรี (Cloudflare ไม่คิดเงินสำหรับ DNS หรือ tunnel)

#### ขั้นที่ 2 — Setup Named Tunnel (ทำครั้งเดียว)

1. เปิด `2-START.bat` ให้เซิร์ฟเวอร์รันก่อน (อย่าปิด)
2. ดับเบิลคลิก `5-NAMED-SETUP.bat`
3. ระบบจะถาม 4 ขั้น:

   **Step 1**: เบราว์เซอร์เปิดให้ login Cloudflare
   - กรอก email + password ของ Cloudflare
   - เลือก domain ที่จะใช้ (เช่น `mycompany.com`)
   - กด **Authorize**
   - ปิดเบราว์เซอร์ กลับมาที่หน้าต่าง bat

   **Step 2**: ตั้งชื่อ tunnel
   - ตัวอย่าง: `hrwwn-companyA`
   - ใช้ตัวเล็ก + ขีดกลาง

   **Step 3**: กรอก URL ที่อยากใช้
   - ตัวอย่าง: `hr.mycompany.com`  หรือ  `app.mycompany.com`
   - อย่าใส่ `https://` นำหน้า
   - ต้องเป็น subdomain ของ domain ที่อยู่บน Cloudflare

   **Step 4**: ระบบเขียน config อัตโนมัติ

4. ขึ้นข้อความ `[OK] เสร็จเรียบร้อย!` แสดงว่าตั้งค่าเสร็จ

#### ขั้นที่ 3 — ใช้งานทุกวัน

ทำเหมือนเดิมเลย:
1. ดับเบิลคลิก `2-START.bat`
2. ดับเบิลคลิก `4-TUNNEL.bat`
3. ระบบจะ **ใช้ named tunnel อัตโนมัติ** (เพราะเจอ config) → URL เป็น `https://hr.mycompany.com`
4. ส่ง URL นั้นให้ลูกค้า — URL **ไม่เปลี่ยน** ทุกครั้งที่รันใหม่

#### ตัวอย่างการใช้งานจริง

```
ที่ออฟฟิศคุณ:
  2-START.bat        → server localhost:3000 รันอยู่
  4-TUNNEL.bat       → tunnel hr.mycompany.com -> localhost:3000

ที่ใดก็ได้ในโลก:
  เข้า https://hr.mycompany.com → ใช้งาน HR-WWN ได้เลย
```

#### ⚠️ ข้อควรระวัง

- **PC ต้องเปิดตลอด** — ถ้าปิด/หลับ ลูกค้าเข้าไม่ได้
- **Internet ของออฟฟิศต้องไม่ขาด** — ถ้าเน็ตล่ม ลูกค้าเข้าไม่ได้
- **กระแสไฟ** — ควรมี UPS ถ้าไฟดับบ่อย
- **ตั้ง Cloudflared เป็น Windows service** ถ้าอยากให้เริ่มเองตอน boot — อ่าน `docs/CLOUDFLARE-TUNNEL.md` หัวข้อ "Auto-start"

## 11. ปัญหาที่พบบ่อย

### Q: ดับเบิลคลิก `2-START.bat` แล้วหน้าต่างปิดทันที
**A**: น่าจะเป็น Node.js ไม่ได้ลง — รัน `1-INSTALL.bat` ก่อน

### Q: เบราว์เซอร์ขึ้น `localhost ปฏิเสธการเชื่อมต่อ`
**A**:
1. ดูที่หน้าต่างสีดำของ `2-START.bat` — มีข้อความ error ไหม?
2. ลองรีเฟรชหน้าเบราว์เซอร์ (F5)
3. ลองเปลี่ยน port: แก้ใน `2-START.bat` บรรทัด `set PORT=3000` เป็น `set PORT=3001`

### Q: Login แล้วขึ้น "username หรือรหัสผ่านไม่ถูกต้อง" — แต่ใส่ถูก
**A**:
- ระวังตัวพิมพ์ใหญ่/เล็ก (case-sensitive)
- ถ้าใส่ผิด 5 ครั้ง ระบบจะ block IP คุณ 15 นาที — รอ 15 นาทีแล้วลองใหม่
- หรือปิด server แล้วเปิดใหม่ (`Ctrl+C` ที่หน้าต่าง 2-START → ดับเบิลคลิกใหม่)

### Q: ลืมรหัส admin
**A**:
1. ปิดเซิร์ฟเวอร์ (`Ctrl+C` ที่ 2-START)
2. ลบไฟล์ `data\auth.json`
3. เปิด `2-START.bat` ใหม่ — ระบบจะสร้าง auth.json ใหม่พร้อมรหัส default `JC2026!Init`
4. Login → เปลี่ยนรหัสเป็นรหัสใหม่ทันที

### Q: ข้อความ Thai ใน `2-START.bat` เป็นสี่เหลี่ยม
**A**: ปกติ — Windows cmd บางเวอร์ชันแสดง Thai ไม่ครบ แต่ระบบทำงานปกติ ไม่กระทบ

### Q: เผลอลบฝ่ายหรือแผนก / อยากย้อนกลับ
**A**:
- ถ้ามี `3-BACKUP.bat` ทำไว้ → คัดลอก data จากใน `backup/` ที่ใหม่สุดทับ
- ถ้าไม่มี backup = ข้อมูลหายถาวร (เลยต้อง backup สม่ำเสมอ!)

### Q: ขึ้นข้อความ "ลบไม่ได้: มี user/แผนก/ตำแหน่ง ผูกอยู่"
**A**: ระบบป้องกันไม่ให้ลบของที่มีคนอยู่ — ต้องลบของในชั้นล่างก่อน
- ลบฝ่าย: ลบแผนก/user ในฝ่ายนั้นก่อน
- ลบแผนก: ลบตำแหน่ง/user ในแผนกนั้นก่อน
- ลบตำแหน่ง: ลบ user + ประวัติ interview ในตำแหน่งนั้นก่อน

## 12. คำถามที่ถามบ่อย (FAQ)

### Q: คอมเปิดทั้งวันได้ไหม กินไฟไหม?
**A**: ได้ — ระบบเบามาก กินไฟเหมือนเปิด Chrome ค้างไว้

### Q: ใช้กี่คนได้?
**A**: เหมาะกับบริษัท 1-200 คน — ไฟล์ JSON ในเครื่องโน้ตบุ๊กระดับกลางรองรับสบาย

### Q: ข้อมูลปลอดภัยแค่ไหน?
**A**:
- รหัสผ่านถูก hash (ดูไม่ได้)
- Cookie HTTP-only + Secure (ถ้าใช้ Cloudflare Tunnel = HTTPS อัตโนมัติ)
- มี rate limit ป้องกัน bot brute-force รหัส
- **แต่ข้อมูลเก็บที่คอมคุณ** — ถ้าคนเข้าถึงคอมได้ก็เปิด data folder ดูได้
- แนะนำ: เปิด BitLocker บน disk ที่เก็บโปรเจกต์

### Q: ปรับฟีเจอร์เพิ่มได้ไหม?
**A**: ได้ แต่ต้องอ่านโค้ด Node.js เป็น — ติดต่อทีม IT หรือคนที่ดูแลโปรแกรมนี้

### Q: ใช้ฟรีตลอดไป?
**A**: ตัวระบบฟรี — ค่าใช้จ่ายมีแค่:
- ไฟ + Internet ของออฟฟิศคุณ
- (ถ้าอยาก) domain name ~$10/ปี สำหรับ URL ถาวร
- (ถ้าอยาก) AI ตอบสนุกขึ้น = ค่า Claude API ~$0.05-0.20 ต่อพนักงาน (ตอนนี้ใช้ AI จำลอง = ฟรี)

### Q: เปลี่ยนเครื่องใหม่ ย้ายข้อมูลยังไง?
**A**:
1. ที่เครื่องเก่า → ดับเบิลคลิก `3-BACKUP.bat`
2. คัดลอกโฟลเดอร์ Hr-wwn ทั้งหมด (รวม `data/` ใน backup ล่าสุด) ไปเครื่องใหม่
3. ที่เครื่องใหม่ → รัน `1-INSTALL.bat` → `2-START.bat`
4. ข้อมูลครบเหมือนเดิม
