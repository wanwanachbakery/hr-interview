# คู่มือการใช้งานระบบ HR-Interview (ฉบับเต็ม)

เอกสารฉบับนี้อธิบายการใช้งานระบบ HR-Interview ทุกส่วนอย่างละเอียด สำหรับผู้ดูแลระบบและผู้ใช้งานทุกบทบาท หากต้องการคู่มือแบบสั้นแยกตามบทบาท ดูได้ที่ "คู่มือฉบับตามบทบาท"

---

## 1. ภาพรวมระบบ

HR-Interview คือระบบบริหารทรัพยากรบุคคลแบบหลายบริษัท (multi-tenant) ที่ช่วยให้องค์กร:

- เก็บโครงสร้างองค์กรเป็นลำดับชั้น: บริษัท → ฝ่าย → แผนก → ตำแหน่ง → พนักงาน
- สัมภาษณ์การทำงานประจำวันของพนักงานแต่ละคน แล้วให้ระบบสร้างเอกสาร 6 ฉบับให้อัตโนมัติ (Job Description, KPI, ข้อเสนอปรับปรุงงาน, ผังงาน ฯลฯ)
- ให้พนักงานบันทึกงานประจำวันรายชั่วโมง และให้หัวหน้าดูภาพรวมของทีม
- สรุปและวิเคราะห์ภาพรวมทั้งองค์กร พร้อมข้อเสนอการนำ AI มาช่วยลดงาน

ระบบรองรับการมีหลายบริษัทในเครื่องเดียว โดยข้อมูลของแต่ละบริษัทแยกขาดจากกันอย่างสมบูรณ์ แต่ละบริษัทมีผู้ดูแล (Admin) ของตนเอง และมีผู้ดูแลระดับสูง (Super Admin) ดูแลภาพรวมการสร้าง/ลบบริษัท

### สิ่งที่ต้องมีก่อนใช้งาน

- เว็บเบราว์เซอร์ (Chrome, Edge, Safari, Firefox) บนคอมพิวเตอร์หรือมือถือ
- ที่อยู่เว็บของระบบ (URL) ที่ผู้ดูแลแจ้งให้
- ชื่อผู้ใช้ (username) และรหัสผ่าน ที่ผู้ดูแลสร้างให้

---

## 2. เริ่มต้นใช้งาน

### 2.1 การเข้าสู่ระบบ

1. เปิดเบราว์เซอร์ไปที่ที่อยู่เว็บของบริษัท เช่น `https://ชื่อเว็บ/t/ชื่อบริษัท/login`
2. กรอก **Username** และ **รหัสผ่าน**
3. กดปุ่ม "เข้าสู่ระบบ"

มีบัญชี 2 ประเภท:

- **ผู้ดูแลระบบ (Admin):** ใช้ username ว่า `admin` และรหัส Master Admin ของบริษัท
- **ผู้ใช้ทั่วไป:** ใช้ username และรหัสผ่านที่ผู้ดูแลสร้างให้ (เช่น `somchai`)

> หากกรอกรหัสผิดหลายครั้งติดกัน ระบบจะหน่วงเวลาชั่วคราวเพื่อความปลอดภัย ให้รอสักครู่แล้วลองใหม่

### 2.2 การออกจากระบบ

กดปุ่ม "ออกจากระบบ" ที่มุมขวาบนของหน้าจอทุกครั้งเมื่อใช้งานเสร็จ โดยเฉพาะเมื่อใช้เครื่องที่ใช้ร่วมกับผู้อื่น

### 2.3 ลืมรหัสผ่าน

- ผู้ใช้ทั่วไป: ติดต่อผู้ดูแลระบบ (Admin) เพื่อให้ตั้งรหัสใหม่
- ผู้ดูแลระบบ: หากลืมรหัส Master Admin ต้องแก้ไขไฟล์ข้อมูลบนเซิร์ฟเวอร์โดยตรง (ดูหัวข้อความปลอดภัย)

## 3. หน้าจอและเมนูหลัก

หลังเข้าสู่ระบบ จะพบหน้าจอที่แบ่งเป็น 3 ส่วน:

- **แถบเมนูด้านซ้าย (Sidebar):** เมนูหลักทั้งหมด พร้อมชื่อบริษัทด้านบน
- **แถบบน (Top bar):** ชื่อหน้าปัจจุบัน ปุ่มโปรไฟล์ และปุ่มออกจากระบบ
- **พื้นที่เนื้อหา:** ส่วนแสดงข้อมูลของหน้าที่เลือก

รายการเมนูในแถบซ้าย (จะแสดงเฉพาะเมนูที่บทบาทของคุณมีสิทธิ์):

| เมนู | ใช้ทำอะไร | ใครเห็น |
|------|-----------|---------|
| หน้าแรก | หน้าเริ่มต้น/ภาพรวมของคุณ | ทุกคน |
| บันทึกงาน | บันทึกงานประจำวันรายชั่วโมงของตนเอง | ทุกคน ยกเว้น Admin |
| บันทึกงานทีม | ดูบันทึกงานของลูกน้องในขอบเขตของตน | หัวหน้าขึ้นไป |
| สรุปบันทึกงาน | รายงานสรุปบันทึกงานของทีม (กราฟ/สัดส่วน) | หัวหน้าขึ้นไป |
| Dashboard | สถานะการสัมภาษณ์ของแต่ละตำแหน่ง | ทุกคน (เห็นตามขอบเขต) |
| รายงาน | สรุปจำนวนผู้ใช้/ฝ่าย + รายชื่อ | ทุกคน (เห็นตามขอบเขต) |
| คู่มือ | เปิดคู่มือการใช้งานนี้ | ทุกคน |
| Admin | ศูนย์จัดการระบบ | Admin เท่านั้น |

> บนมือถือ แถบเมนูซ้ายจะถูกซ่อน กดปุ่ม ☰ (สามขีด) ที่มุมซ้ายบนเพื่อเปิดเมนู

## 4. บทบาทผู้ใช้และสิทธิ์การเข้าถึง

ระบบมี 6 บทบาทหลัก เรียงจากสิทธิ์มากไปน้อย และมีผู้ดูแลระดับสูงอีก 1 ระดับ หลักการคือ "ยิ่งตำแหน่งสูง ยิ่งเห็นข้อมูลกว้าง"

| บทบาท | ขอบเขตการมองเห็น |
|-------|------------------|
| ผู้ดูแลระบบ (admin) | เห็นและจัดการได้ทั้งบริษัท + ตั้งค่าระบบ |
| ผู้บริหาร (executive) | เห็นข้อมูลทั้งบริษัท (อ่าน) |
| ผู้จัดการ (manager) | เห็นเฉพาะฝ่ายของตน + ที่ผู้ดูแลอนุญาตเพิ่ม |
| หัวหน้าฝ่าย (division_head) | เห็นเฉพาะฝ่ายของตน |
| หัวหน้าแผนก (section_head) | เห็นเฉพาะแผนกของตน |
| เจ้าหน้าที่ (officer) | เห็นเฉพาะข้อมูลของตนเอง |
| ผู้ดูแลระดับสูง (super admin) | จัดการการสร้าง/ลบบริษัท (ไม่ยุ่งข้อมูลภายในบริษัท) |

หลักสำคัญ:

- **การดู (อ่าน):** หัวหน้าเห็นข้อมูลของลูกน้องในสายงานของตนได้
- **การแก้ไข (เขียน):** ตอบ/ปิดการสัมภาษณ์ของตนเองได้เท่านั้น (Admin แก้ไขแทนได้ในกรณีจำเป็น)
- พนักงานทั่วไป (officer) เห็นและแก้ไขเฉพาะของตนเอง

---

## 5. โครงสร้างองค์กร

ระบบจัดข้อมูลเป็นลำดับชั้น 4 ระดับ ต้องสร้างจากบนลงล่าง:

1. **ฝ่าย (Division)** — หน่วยใหญ่สุด เช่น ฝ่ายขาย ฝ่ายบัญชี
2. **แผนก (Section)** — อยู่ภายใต้ฝ่าย เช่น แผนกขายหน้าร้าน
3. **ตำแหน่ง (Position)** — อยู่ภายใต้แผนก เช่น พนักงานขาย
4. **พนักงาน (User)** — ผู้ใช้ที่ครองตำแหน่งนั้น

แนวคิดสำคัญ: ระบบยึด "ตำแหน่ง" เป็นหลัก เมื่อสร้างผู้ใช้และกำหนดตำแหน่งให้ ระบบจะสร้างระเบียนพนักงานสำหรับตำแหน่งนั้นอัตโนมัติ ทำให้ติดตามได้ว่าใครครองตำแหน่งใด และตำแหน่งใดยังว่าง

---

## 6. งานของผู้ดูแลระบบ (Admin)

เข้าเมนู "Admin" (แสดงเฉพาะผู้ดูแล) จะพบศูนย์จัดการระบบ

### 6.1 หน้า Admin Console

หน้านี้รวมทางลัดไปยังงานจัดการทั้งหมด: โครงสร้างองค์กร, จัดการผู้ใช้, Dashboard, รายงาน, รหัส Master Admin, การตั้งค่า Claude AI, Danger Zone และข้อมูลโดยสรุป

### 6.2 จัดการโครงสร้างองค์กร

ไปที่ "โครงสร้างองค์กร" เพื่อสร้าง/แก้ไข/ลบ ฝ่าย แผนก และตำแหน่ง

ขั้นตอนแนะนำเมื่อเริ่มระบบใหม่:

1. สร้าง **ฝ่าย** ทั้งหมดก่อน (ตั้งชื่อ ไอคอน และสี)
2. สร้าง **แผนก** ในแต่ละฝ่าย
3. สร้าง **ตำแหน่ง** ในแต่ละแผนก
4. จากนั้นจึงไปสร้างผู้ใช้แล้วผูกกับตำแหน่ง

### 6.3 จัดการผู้ใช้

ไปที่ "จัดการผู้ใช้" เพื่อเพิ่ม/แก้ไข/ลบผู้ใช้

การเพิ่มผู้ใช้:

1. กดเพิ่มผู้ใช้ใหม่
2. กรอก ชื่อ-นามสกุล, username, รหัสผ่านเริ่มต้น
3. เลือก **บทบาท (role)**
4. เลือก **ฝ่าย / แผนก / ตำแหน่ง** ที่สังกัด (จำเป็นต้องครบทั้ง 3 ระดับ)
5. กำหนดเวลาทำงาน (เริ่ม-เลิก และเวลาพัก) — ใช้เป็นช่วงชั่วโมงในหน้าบันทึกงาน
6. บันทึก

ข้อกำหนด:

- ห้ามใช้ username ว่า `admin` (สงวนไว้สำหรับผู้ดูแล)
- ผู้ใช้ทุกคน (ยกเว้น admin) ต้องมีครบทั้งฝ่าย/แผนก/ตำแหน่ง
- เมื่อสร้างผู้ใช้พร้อมตำแหน่ง ระบบจะสร้างระเบียนพนักงานให้อัตโนมัติ
- เมื่อแก้ไขผู้ใช้ จะเปลี่ยน role/ตำแหน่งได้เฉพาะผู้ดูแล

### 6.4 นำเข้าข้อมูลด้วย Excel

ระบบรองรับการสร้างข้อมูลจำนวนมากด้วยไฟล์ Excel:

1. ดาวน์โหลด **เทมเพลต** (มีทั้งเทมเพลตโครงสร้างองค์กร และเทมเพลตผู้ใช้)
2. กรอกข้อมูลตามคอลัมน์ — มีชีต "ค่าที่ใช้ได้" แสดงรายการฝ่าย/แผนก/ตำแหน่งปัจจุบันให้อ้างอิง
3. อัปโหลดไฟล์กลับเข้าระบบ
4. ระบบจะรายงานผลและแจ้งแถวที่มีปัญหา (ถ้ามี)

> แนะนำให้สร้างโครงสร้างองค์กรก่อน แล้วจึงนำเข้าผู้ใช้ เพราะผู้ใช้ต้องอ้างอิงฝ่าย/แผนก/ตำแหน่งที่มีอยู่แล้ว

### 6.5 รหัส Master Admin

ใช้เปลี่ยนรหัสผ่านของผู้ดูแลระบบ (บัญชี `admin`)

- รหัสปัจจุบันถูกเข้ารหัสไว้ (ดูค่าเดิมไม่ได้)
- ตั้งรหัสใหม่ได้ ความยาวอย่างน้อย 6 ตัวอักษร
- หากลืมรหัส ต้องแก้ไฟล์ข้อมูลบนเซิร์ฟเวอร์โดยตรง

### 6.6 การตั้งค่า Claude AI

ส่วนนี้ควบคุมการสร้างเอกสารด้วยปัญญาประดิษฐ์ (Claude)

- **สวิตช์เปิด/ปิด:** เปิด = ใช้ AI วิเคราะห์จริงตอนกด "เสร็จสิ้นสัมภาษณ์" และ "วิเคราะห์ภาพรวมองค์กร"; ปิด = ใช้เทมเพลตสำเร็จรูป (ฟรี ไม่มีค่าใช้จ่าย)
- การเปิด/ปิดต้องกรอก **รหัส Admin** ยืนยันทุกครั้ง
- **สถานะ API Key:** บอกว่าได้ตั้งค่ากุญแจ API บนเซิร์ฟเวอร์แล้วหรือยัง หากยังไม่ตั้ง แม้เปิดสวิตช์ ระบบก็จะยังใช้เทมเพลตจนกว่าจะตั้งกุญแจ
- **การใช้งานสะสม:** แสดงจำนวนครั้งที่วิเคราะห์ จำนวนโทเคน (token) เข้า-ออก และค่าใช้จ่ายโดยประมาณ (บาท/ดอลลาร์) เฉพาะของบริษัทนี้
- **ประวัติการวิเคราะห์ล่าสุด:** ตารางแสดงรายครั้งว่าใช้โทเคนและค่าใช้จ่ายเท่าใด
- **ปุ่มล้างตัวนับ:** ล้างยอดสะสมและประวัติ

> ค่าใช้จ่ายเป็นการประมาณจากอัตราค่าโทเคนของโมเดล และแปลงเงินบาทโดยประมาณ ใช้เพื่อดูแนวโน้มการใช้งาน

### 6.7 Danger Zone (ลบข้อมูล)

โซนอันตราย — ใช้ลบข้อมูลเป็นกลุ่ม ทุกปุ่มต้องพิมพ์คำว่า `DELETE` ยืนยัน และลบแล้วกู้คืนไม่ได้

1. **ลบ User ทั้งหมด** — เคลียร์ผู้ใช้ทั้งหมด (เก็บโครงสร้างองค์กรและประวัติการสัมภาษณ์)
2. **ลบ ฝ่าย/แผนก/ตำแหน่ง** — เคลียร์โครงสร้างองค์กร และลบผู้ใช้/พนักงาน/การสัมภาษณ์ที่อ้างอิงด้วย
3. **ลบข้อมูลที่ User กรอก** — เคลียร์เฉพาะคำตอบสัมภาษณ์ + เอกสารที่สร้าง (รีเซ็ตสถานะกลับเป็น "ยังไม่ตอบ")
4. **ลบข้อมูลทั้งหมด** — รีเซ็ตทั้งระบบของบริษัท เหลือเพียงรหัสผู้ดูแล

> สำรองข้อมูลก่อนทุกครั้งก่อนใช้ปุ่มในโซนนี้

### 6.8 ข้อมูลโดยสรุป

แสดงภาพรวมของบริษัทแบบรายบรรทัด: ชื่อบริษัท จำนวนฝ่าย จำนวนแผนก จำนวนตำแหน่ง และจำนวนผู้ใช้

## 7. Dashboard — สถานะการสัมภาษณ์

หน้านี้แสดงโครงสร้างองค์กรเป็นต้นไม้ (ฝ่าย → แผนก → ตำแหน่ง) พร้อมสถานะการสัมภาษณ์ของผู้ครองแต่ละตำแหน่ง:

- **ยังไม่ตอบ** — ยังไม่เริ่มสัมภาษณ์
- **กำลังตอบ** — ตอบอยู่ ยังไม่เสร็จ
- **เสร็จ** — ตอบครบและสร้างเอกสารแล้ว (มีลิงก์ "ดูเอกสาร")
- **ตำแหน่งว่าง** — ยังไม่มีผู้ครองตำแหน่ง

ความสามารถเพิ่มเติม:

- **ปุ่มวิเคราะห์ภาพรวมบริษัท** (ผู้ดูแล/ผู้บริหาร/ผู้จัดการ) — สร้างรายงาน Optimization รวมทั้งองค์กร
- **ปุ่มรีเซ็ต** (ผู้ดูแล) — ล้างคำตอบและเอกสารของคนนั้น กลับเป็น "ยังไม่ตอบ" โดยไม่ลบตัวพนักงาน เพื่อให้กรอกใหม่ได้

---

## 8. การสัมภาษณ์งาน (Interview)

การสัมภาษณ์คือการให้พนักงานเล่างานประจำวันของตน ระบบจะถามทีละช่วงเวลา

ขั้นตอน:

1. จาก Dashboard หรือหน้าแรก กดเริ่ม/ตอบการสัมภาษณ์ของตน
2. ตอบคำถามตามช่วงชั่วโมงและหัวข้อ (งานหลัก เครื่องมือที่ใช้ คนที่ประสานงาน ปัญหา/คอขวด งานรายสัปดาห์ ฯลฯ)
3. เมื่อตอบครบ กดปุ่ม "เสร็จสิ้น"
4. จะมีหน้าต่างแจ้ง "กำลังวิเคราะห์..." จนขึ้น "วิเคราะห์สำเร็จ" จากนั้นไปดูเอกสารได้

> ขณะกดเสร็จสิ้น สามารถเลือกให้ระบบดึง "บันทึกงานประจำวันจริง" ย้อนหลัง 1/2/3 เดือนมาประกอบการวิเคราะห์ได้ ทำให้เอกสารตรงกับงานจริงมากขึ้น

---

## 9. เอกสารที่ระบบสร้างให้ (6 ฉบับ)

เมื่อสัมภาษณ์เสร็จ ระบบสร้างเอกสาร 6 ฉบับให้ ดูได้ที่หน้า "ดูเอกสาร" ของพนักงาน (มีปุ่มกระโดดไปแต่ละฉบับ และปุ่ม "เปิดไฟล์" เพื่อดาวน์โหลด)

1. **Job Description** — ใบกำหนดหน้าที่งานจากงานจริง
2. **KPI** — ตัวชี้วัดผลงานที่วัดได้
3. **Optimization** — ข้อเสนอปรับปรุงงาน และการนำ AI มาช่วย
4. **Workflow** — ขั้นตอนงานประจำวันแบบละเอียด
5. **ผังงาน (Flowchart)** — แผนภาพลำดับงาน (นำโค้ดไปวาดที่ mermaid.live ได้)
6. **ตารางงาน (CSV)** — เปิดใน Excel ได้

## 10. การวิเคราะห์ภาพรวมองค์กร

ผู้ดูแล/ผู้บริหาร/ผู้จัดการ กดปุ่ม "วิเคราะห์ภาพรวมบริษัท" ใน Dashboard เพื่อให้ระบบรวมข้อมูลของทุกคนที่สัมภาษณ์เสร็จ แล้วสร้าง **รายงาน Optimization ระดับองค์กร** ครอบคลุม:

- ความครอบคลุมรายฝ่าย (สัมภาษณ์ไปกี่คน)
- ปัญหา/คอขวดที่พบซ้ำหลายคน
- โอกาสนำ AI/automation มาช่วย จัดลำดับตามผลกระทบ
- ข้อเสนอภาพรวมและแผนระยะสั้น-ยาว

รายงานล่าสุดจะแสดงในหน้า Dashboard และเปิด/ดาวน์โหลดเป็นไฟล์ได้

## 11. บันทึกงานประจำวัน

เมนู "บันทึกงาน" ให้พนักงานบันทึกสิ่งที่ทำในแต่ละชั่วโมงของวัน

วิธีใช้:

1. เลือกวันที่ (ค่าเริ่มต้นคือวันนี้)
2. ในแต่ละช่วงชั่วโมง พิมพ์งานที่ทำ และเลือกหมวดหมู่
3. กด **"+ เพิ่มงานในชั่วโมงนี้"** เพื่อใส่ได้หลายงานใน 1 ชั่วโมง / กด ✕ เพื่อลบงานนั้น
4. ติ๊ก **"งานประจำ"** สำหรับงานที่ทำซ้ำทุกวัน
5. กด **"คัดลอกงานประจำจากเมื่อวาน"** เพื่อดึงงานประจำของเมื่อวานมาเติม
6. กด **"บันทึกงานวันนี้"**

คุณสมบัติเสริม:

- **หมวดหมู่:** ผู้ดูแลตั้งชุดเริ่มต้นให้ และพนักงานเพิ่มหมวดหมู่ใหม่เองได้จากช่องเลือกหมวดหมู่
- **แถบความคืบหน้า:** แสดงว่าบันทึกไปแล้วกี่ชั่วโมงจากชั่วโมงทำงานทั้งหมด
- **การแจ้งเตือนในแอป:** หากยังไม่ได้บันทึกงานของวันนี้ จะมีข้อความเตือนที่หน้าแรก/Dashboard

---

## 12. บันทึกงานของทีม

เมนู "บันทึกงานทีม" (สำหรับหัวหน้าขึ้นไป) ใช้ดูบันทึกงานของลูกน้องในขอบเขตของตน

วิธีใช้:

1. เลือกพนักงานจากช่อง **ดรอปดาวน์** (ค้นหาด้วยการพิมพ์ชื่อได้)
2. คัดกรองตามฝ่าย/แผนกได้
3. เลือกวันที่ที่ต้องการดู
4. ระบบแสดงตารางงานรายชั่วโมงของคนนั้น (อ่านอย่างเดียว แก้ไม่ได้)

> เจ้าหน้าที่ (officer) จะไม่เห็นเมนูนี้ เพราะดูได้เฉพาะของตนเอง

---

## 13. สรุปบันทึกงาน

เมนู "สรุปบันทึกงาน" (สำหรับหัวหน้าขึ้นไป) แสดงรายงานสรุปบันทึกงานของทีม:

- เลือกช่วงเวลา (รายสัปดาห์/รายเดือน หรือกำหนดช่วงเอง)
- กราฟ/สัดส่วนเวลาที่ใช้แยกตามหมวดหมู่งาน
- จำนวนงานทั้งหมด และสัดส่วนงานประจำ (งานซ้ำ)
- รายชื่อผู้ที่บันทึกครบ/ไม่ครบ

ใช้เพื่อดูว่าทีมใช้เวลาไปกับงานประเภทใดมากที่สุด และงานซ้ำส่วนไหนที่ควรลด/นำ AI มาช่วย

---

## 14. รายงาน

เมนู "รายงาน" แสดงสรุปข้อมูลผู้ใช้ตามขอบเขตของบทบาทคุณ:

- **สรุปภาพรวม:** จำนวนผู้ใช้ทั้งหมด จำนวนฝ่าย และจำนวนแยกตามบทบาท
- **จำนวนผู้ใช้แต่ละฝ่าย:** ตารางจำนวนคนต่อฝ่าย
- **รายชื่อผู้ใช้:** ค้นหาด้วยชื่อ/username และคัดกรองตาม role/ฝ่าย/แผนก พร้อมเวลาทำงานของแต่ละคน

---

## 15. โปรไฟล์ส่วนตัว

เมนู "โปรไฟล์" (มุมขวาบน) ให้ผู้ใช้ดูข้อมูลของตนเองและเปลี่ยนรหัสผ่านส่วนตัว (ผู้ดูแลไม่มีโปรไฟล์ส่วนตัว เพราะใช้รหัส Master Admin)

## 16. ผู้ดูแลระดับสูง (Super Admin)

ผู้ดูแลระดับสูงเข้าที่ `https://ชื่อเว็บ/super/login` ด้วยรหัส Super Admin

หน้าที่:

- สร้างบริษัทใหม่ (กำหนดรหัสบริษัท ชื่อบริษัท และรหัส Admin เริ่มต้น)
- ดูรายชื่อบริษัททั้งหมด
- ลบบริษัท

> Super Admin ไม่ยุ่งกับข้อมูลภายในบริษัท การจัดการผู้ใช้/โครงสร้าง/การสัมภาษณ์ เป็นหน้าที่ของ Admin แต่ละบริษัท

## 17. ความปลอดภัยและการสำรองข้อมูล

- **เปลี่ยนรหัสเริ่มต้น:** เปลี่ยนรหัส Super Admin และ Master Admin ทันทีหลังติดตั้ง
- **สำรองข้อมูลสม่ำเสมอ:** สำรองโฟลเดอร์ข้อมูลก่อนทำการลบใด ๆ ใน Danger Zone
- **กุญแจ API (Claude):** เก็บเป็นค่าตั้งค่าบนเซิร์ฟเวอร์เท่านั้น ไม่ใส่ในโค้ด และห้ามเผยแพร่ หากหลุดให้เพิกถอนและออกกุญแจใหม่
- **ออกจากระบบทุกครั้ง:** โดยเฉพาะบนเครื่องที่ใช้ร่วมกัน
- **รหัสผ่าน:** ตั้งให้คาดเดายาก อย่างน้อย 6 ตัวอักษร

## 18. คำถามที่พบบ่อย (FAQ)

**ถาม: เข้าระบบไม่ได้ ขึ้นว่ารหัสผิด**
ตอบ: ตรวจ username/รหัสให้ถูก หากผิดหลายครั้งระบบจะหน่วงเวลา ให้รอสักครู่ หากยังไม่ได้ให้ติดต่อผู้ดูแล

**ถาม: บันทึกงานหรือเอกสารหายหลังกดลบ**
ตอบ: ปุ่ม "รีเซ็ต" ใน Dashboard จะล้างคำตอบ/เอกสารและกลับเป็น "ยังไม่ตอบ" แต่ไม่ลบตัวพนักงาน สามารถกรอกใหม่ได้

**ถาม: หน้าตาเว็บไม่อัปเดตหลังปรับปรุงระบบ**
ตอบ: กด Ctrl + Shift + R หนึ่งครั้งเพื่อล้างแคชเบราว์เซอร์

**ถาม: เปิด Claude แล้วทำไมยังขึ้นว่าใช้เทมเพลต (Mock)**
ตอบ: ต้องตั้งค่ากุญแจ API บนเซิร์ฟเวอร์ก่อน เมื่อตั้งแล้วและเปิดสวิตช์ ระบบจะใช้ Claude ทันที

**ถาม: ทำไมเห็นข้อมูลไม่ครบเท่าคนอื่น**
ตอบ: ระบบแสดงข้อมูลตามขอบเขตของบทบาท ยิ่งตำแหน่งสูงยิ่งเห็นกว้าง หากต้องการสิทธิ์เพิ่มให้ติดต่อผู้ดูแล
