# คู่มือการใช้งาน HR-Interview

ระบบเก็บข้อมูลการทำงานของพนักงาน → สร้างเอกสาร HR อัตโนมัติ + บันทึกงานประจำวัน + รายงานสรุป
รองรับ **หลายบริษัทบนระบบเดียว** (multi-tenant) · เว็บ: **https://wanwanachapp.com**

---

## 1. ภาพรวมระบบ

ระบบมี 3 ส่วนงานหลัก:

1. **อินเทอร์วิว → เอกสาร HR** — พนักงานตอบคำถามการทำงานรายชั่วโมง ระบบสร้างให้อัตโนมัติ:
   - Job Description (ใบกำหนดหน้าที่)
   - KPI (ตัวชี้วัดผลงาน)
   - Optimization (ข้อเสนอลดงาน/ใช้ AI)
   - Workflow (ผังงาน + ตาราง)
2. **บันทึกงานประจำวัน (Worklog)** — พนักงานบันทึกงานที่ทำแต่ละชั่วโมงทุกวัน พร้อมหมวดหมู่และธง"งานประจำ"
3. **รายงาน/วิเคราะห์** — หัวหน้าดูบันทึกของทีม + รายงานสรุปภาพรวม

> 🤖 การสร้างเอกสารใช้ **Claude (Sonnet 4.6)** เมื่อเปิดใช้งาน · ถ้าปิดจะใช้ตัวสร้างแม่แบบสำรอง (ระบบทำงานได้ทั้งสองแบบ)

---

## 2. บทบาทผู้ใช้ (สิทธิ์การมองเห็น)

| บทบาท | เห็นข้อมูลใคร |
|---|---|
| **Super Admin** | ทุกบริษัท (ดูแลระบบสูงสุด) |
| **Admin** | ทุกอย่างใน 1 บริษัทของตน |
| **Executive (ผู้บริหาร)** | ทุกฝ่ายในบริษัท |
| **Division Head (หัวหน้าฝ่าย)** | ทั้งฝ่ายของตน |
| **Section Head (หัวหน้าแผนก)** | ทั้งแผนกของตน |
| **Manager (ผู้จัดการ)** | ตามขอบเขตที่ admin กำหนด |
| **Officer (เจ้าหน้าที่)** | เฉพาะของตัวเอง |

**โครงสร้างองค์กร:** บริษัท → ฝ่าย (division) → แผนก (section) → ตำแหน่ง (position) → ผู้ใช้ผูกกับตำแหน่ง

---

## 3. การเข้าสู่ระบบ

| ใคร | ลิงก์ | บัญชี |
|---|---|---|
| Super Admin | `https://wanwanachapp.com/super/login` | รหัสผ่าน super |
| Admin บริษัท | `https://wanwanachapp.com/t/<รหัสบริษัท>/login` | username `admin` |
| พนักงาน/หัวหน้า | `https://wanwanachapp.com/t/<รหัสบริษัท>/login` | username + รหัสผ่านที่ admin ตั้งให้ |

---

## 4. วิธีใช้ — Super Admin (ผู้ดูแลระบบสูงสุด)

หน้า `/super/login`

1. **สร้างบริษัทใหม่** — กรอกรหัสบริษัท (a-z, 0-9), ชื่อบริษัท, รหัส admin เริ่มต้น → ระบบสร้างให้พร้อมใช้
2. **เปลี่ยนชื่อบริษัท** — แก้แล้วชื่อจะอัปเดตในบริษัทนั้นทันที
3. **รีเซ็ตรหัส admin** ของบริษัทใด ๆ
4. **ลบบริษัท** (พิมพ์ยืนยัน DELETE — ลบข้อมูลทั้งหมดของบริษัทนั้น ระวัง!)
5. **เปลี่ยนรหัส super-admin** ของตัวเอง

---

## 5. วิธีใช้ — Admin (ผู้ดูแลบริษัท)

เข้า `/t/<บริษัท>/login` ด้วย username `admin` → เมนู **🔐 Admin**

1. **จัดโครงสร้างองค์กร** (Admin → จัดการองค์กร): เพิ่ม ฝ่าย → แผนก → ตำแหน่ง
   - หรือ **นำเข้าด้วย Excel** (มีไฟล์เทมเพลต + ชีตข้อมูลอ้างอิงให้กรอกไม่ผิด)
2. **เพิ่มผู้ใช้** (Admin → จัดการผู้ใช้): ตั้ง username, รหัสผ่าน, ชื่อ, บทบาท, ฝ่าย/แผนก/ตำแหน่ง, เวลาทำงาน
   - หรือนำเข้าด้วย Excel
3. **แก้ชื่อบริษัท / ข้อมูลบริษัท**
4. **ดูทุกอย่าง**: dashboard, รายงาน, บันทึกงานทุกคน, สร้างรายงานภาพรวมบริษัท
5. **Danger Zone** (ลบข้อมูล): ลบ user / ลบตำแหน่ง / ลบข้อมูลที่กรอก / ล้างทั้งหมด — ต้องยืนยัน
6. **ตั้งหมวดหมู่งานเริ่มต้น** — ระบบมีชุดเริ่มต้นให้ พนักงานเพิ่มเองได้ภายหลัง

---

## 6. วิธีใช้ — หัวหน้า / ผู้บริหาร

นอกจากทำของตัวเองได้เหมือนพนักงาน ยังมีเมนูเพิ่ม:

- **👥 บันทึกงานทีม** — ค้นหา/เลือกพนักงานในขอบเขตของตน → ดูบันทึกงานรายชั่วโมงของเขา (อ่านอย่างเดียว) · คัดกรองตาม ฝ่าย/แผนก/สถานะ + เลือกวัน
- **📈 สรุปบันทึกงาน** — เลือกช่วงเวลา (สัปดาห์/เดือน/กำหนดเอง) ดู:
  - ความครบเฉลี่ย · ชั่วโมงรวม · **% งานซ้ำ** · จำนวนคนที่บันทึก
  - กราฟ % เวลาแยกตามหมวดหมู่ + ตารางอัตราการบันทึกรายคน (ดี/พอใช้/ต่ำ)
  - **ดาวน์โหลด CSV** ได้
- เห็นเฉพาะคนในขอบเขตสิทธิ์ (หัวหน้าแผนกเห็นแค่แผนกตน ฯลฯ)

## 7. วิธีใช้ — พนักงาน (Officer)

### 7.1 อินเทอร์วิว (ทำครั้งเดียว/เมื่อมีการอัปเดต)
1. หน้าแรก → **เริ่มอินเทอร์วิวของฉัน** → ยืนยันเวลาทำงาน + วันที่
2. ตอบคำถามการทำงานรายชั่วโมงตามที่ระบบถาม
3. กด **สร้างเอกสาร** → เลือก "ใช้ข้อมูลบันทึกงานย้อนหลัง" (1/2/3 เดือน หรือไม่ใช้)
4. รอป็อปอัพ "กำลังวิเคราะห์ → สำเร็จ" → ระบบพาไปดูเอกสาร JD/KPI/Optimization

### 7.2 บันทึกงานประจำวัน (📝 บันทึกงาน) — ทำทุกวัน
1. ระบบสร้างตารางรายชั่วโมงตามเวลาทำงานของคุณ (ข้ามพักเที่ยงอัตโนมัติ)
2. แต่ละชั่วโมง: กรอกงานที่ทำ + เลือก **หมวดหมู่** (เลือกจากรายการ หรือ "+ เพิ่มหมวดหมู่ใหม่")
3. ติ๊ก **⟳ งานประจำ** สำหรับงานที่ทำซ้ำทุกวัน
4. ปุ่ม **📋 คัดลอกงานประจำจากเมื่อวาน** — ดึงงานประจำมาใส่ ไม่ต้องพิมพ์ซ้ำ
5. กด **บันทึกงานวันนี้** · เลือกวันย้อนหลังเพื่อแก้วันก่อนได้
6. หน้าแรกจะเตือนถ้ายังบันทึกวันนี้ไม่ครบ (ปิดการเตือนได้ในหน้าบันทึกงาน)

### 7.3 โปรไฟล์ (👤 โปรไฟล์)
- แก้เวลาทำงานของตัวเอง · เปลี่ยนรหัสผ่าน

## 8. การดูแลระบบ (สำหรับ IT)

ระบบรันบน **DigitalOcean Droplet + PM2 + Cloudflare Tunnel** · โดเมน wanwanachapp.com

| งาน | คำสั่ง (บน droplet) |
|---|---|
| อัปเดตโค้ดใหม่ | `cd /opt/hr-interview && git pull && npm install --omit=dev && pm2 restart hr-interview` |
| ดูสถานะ | `pm2 status` · `systemctl status cloudflared` |
| ดู log | `pm2 logs hr-interview` |
| เปิด Claude | `export ANTHROPIC_API_KEY=sk-ant-... ; pm2 restart hr-interview --update-env ; pm2 save` |
| ปิด Claude | `pm2 delete hr-interview ; PORT=3000 NODE_ENV=production SECURE_COOKIES=true AUTO_OPEN_BROWSER=false pm2 start server.js --name hr-interview ; pm2 save` |
| Backup รายวัน (cron ตี 2) | `mkdir -p /root/backups; (crontab -l 2>/dev/null; echo "0 2 * * * tar czf /root/backups/hr-\$(date +\%F).tar.gz /opt/hr-interview/data") \| crontab -` |

> ข้อมูลทั้งหมดอยู่ในโฟลเดอร์ `/opt/hr-interview/data/` (git ไม่แตะ) — backup โฟลเดอร์นี้พอ

## 9. รหัสเริ่มต้น & ความปลอดภัย (เปลี่ยนก่อนใช้จริง!)

- รหัส Super Admin เริ่มต้น: `super!2026` → **เปลี่ยนทันที**
- รหัส Admin บริษัทใหม่ (ค่าเริ่มต้น): `WWN2026!Init` → เปลี่ยนหลังสร้าง
- **API key ของ Claude**: เก็บเป็น env var บน server เท่านั้น — ห้ามใส่ในโค้ด/ห้ามขึ้น GitHub
- ลบบริษัทเดโม/ข้อมูลทดสอบก่อนเปิดใช้จริง

## 10. ขีดความสามารถ (จากการทดสอบจริง)

ทดสอบแล้วที่ **20 บริษัท · 520 คน · ข้อมูลย้อนหลัง ~3 ปี (379,600 ระเบียน)**:
- ทุกฟังก์ชันทำงานถูกต้อง · แยกข้อมูลข้ามบริษัทแน่นหนา
- หน้ารายงานหนักสุด ~0.5 วินาที · บันทึกงานรายคน ~0.003 วินาที
- รองรับลักษณะ "หลายบริษัท บริษัทละ 20-30 คน" ได้สบาย
- ข้อควรเผื่อ: ถ้าบริษัทเดียวมีพนักงานหลายร้อยคน หน้ารายงานจะเริ่มช้า (แก้ได้ภายหลังด้วยการปรับรูปแบบจัดเก็บ)
